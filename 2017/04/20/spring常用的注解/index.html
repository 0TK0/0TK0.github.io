spring常用的注解

By zhuyuqing

发表于 2017-04-20

文章目录
  1. 1. swagger相关
  • @RestController：相当于@Controller + @ResponseBody 可以直接返回json ——spring mvc
  • @SpringBootApplication：相当于 -> @Configuration + @EnableAutoConfiguration + @ComponentScan
  • @Configuration:指明该类由spring容器管理
  • @EnableAutoConfiguration：无xml配置启动的关键部分 ——spring boot
  • @ComponentScan：指定扫描包（如果主类不在根路径下，这里需要指定扫描路径），类似于spring的包扫描注解
  • @ConfigurationProperties（prefix=“user”）：自动读取application.properties（是spring-boot默认查找的文件）文件中user.*的属性，并根据成员变量名一一匹配赋值
  • @Value（”${user.id}”)：在没有用@ConfigurationProperties注解的前提下，从application.properties一个个指定属性的值
  • @Bean:配合@Configuration（类级别）用在方法上，
  • 方法的返回值为单例
  • 方法返回值可以直接注入到其他类中使用
  • 方法级别的
  • @Component:
  • 方法返回的是一个多例
  • 返回值不能直接注入到其他类中使用
  • 类级别
  • @MapperScan（basePackage=“com.xxx.mapper”）：指定扫描的mapper接口所在的包,此处mapper接口在com.xxx.mapper下。
  • @Primary：指定在同一个接口有多个实现类可以注入的时候，默认选择被它注解的，而不是让@Autowire注解报错（一般用于多数据源的情况下）
  • @Transactional：添加事务，一般写在service层的方法上面

swagger相关

  • @Api：用在类上，说明该类的作用
  • @ApiOperation：用在方法上，说明方法的作用
  • @ApiImplicitParams：用在方法上包含一组参数说明
  • @ApiImplicitParam：用在@ ApiImplicitParams注释中，指定一个请求参数的各个方面
  • paramType：参数放在哪个地方
    • header -> 请求参数的获取：@RequestHeader
    • query -> 请求参数的获取：@RequestParam
    • path （用于restful接口） -> 请求参数的获取：@PathVariable
    • body（不常用）
    • form（不常用）
    • name：参数名
  • dataType：参数类型
  • required：参数是否必须传
  • value：参数的意思
  • defaultValue：参数的默认值
  • @ApiResponses：用于表示一组响应
  • @ApiResponse：用在@ ApiResponses中，一般用于表达一个错误的响应信息
  • code：数字，例如400
  • message：信息，例如“请求参数没填好”
  • response：抛出异常的类
  • ApiModel：描述一个Model的信息（这种一般用在post创建的时候，使用@RequestBody这样的场景，请求参数无法使用@ApiImplicitParam注解进行描述的时候）
  • /常用的关于swagger的注解就是上面这些，其他注解，查看 Annotations · swagger-api/swagger-core Wiki · GitHub /

@ControllerAdvice：是controller的辅助类，常用做全局异常处理的切面类，可以指定扫描范围@ControllerAdvice(basePackages={“com.xxx”,”com.ooo”})
如果直接返回model类需要加上@ResponseBody注解。返回类型：返回String，跳到view；返回modelAndView；返回model+@ResponseBody

@PostConstruct:容器初始化bean
@PreDestroy:容器销毁bean之前操作
